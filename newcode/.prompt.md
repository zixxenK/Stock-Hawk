# Flippy Engine wiring prompt

## Purpose
This prompt helps an AI agent wire the full Flippy Engine pipeline from raw alternative data ingestion through database persistence, feature vector construction, agent scoring, and API exposure.

## Instructions
- Read the repository guidance in `AGENTS.md` and the live ingestion / broker execution guidance in `SKILL.md` first.
- Preserve the existing module separation: ingestion under `data_sources/`, persistence under `database/`, signal processing under `processing/`, intelligence under `intelligence/`, and API serving under `api/`.
- Do not introduce or leave any `pass`, `...`, or `# TODO` in structural code.
- Use Python 3.10+ type annotations and Pydantic v2+ schemas where raw data enters the system.
- Ensure all database adapters use SQLite WAL mode by executing `PRAGMA journal_mode=WAL` on initialization.
- Keep `rl_insider_trader.py` as the Gymnasium environment contract and do not leak ingestion logic into it.
- Render observation vectors as `np.array(..., dtype=np.float32)` with the expected 11-element shape when used by RL components.

## Task
1. Wire the ingestion sources through `data_sources/scraper_manager.py` to the persistence layer in `database/db_manager.py`.
2. Ensure scraper raw responses are saved before parsing and persist structured records afterward.
3. Build the feature vector in `processing/vectorizer.py` from the cleaned insider/congress/sentiment inputs.
4. Wire `intelligence/agent.py` to receive the feature vector and return a concrete action recommendation and alpha score.
5. Ensure `api/fast_api_app.py` exposes working endpoints for single ticker analysis, batch analysis, backtest, and history retrieval.
6. Ensure `main.py` can run the full golden loop without import errors and without depending on stale legacy packaging.

## Output
- Update or create code so the entire program is wired and runnable.
- Keep the implementation robust and production-ready.
- After wiring, validate by importing the main entrypoint and the database manager.
